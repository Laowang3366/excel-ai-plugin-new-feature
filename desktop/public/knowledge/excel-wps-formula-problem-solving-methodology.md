# Excel/WPS 公式解题方法论

## 这份知识如何使用

这份知识由既有公式案例归纳而来，目标不是寻找一个最相似案例并替换单元格地址，而是先识别问题结构，再组合通用变换，最后验证结果。

处理公式任务时遵循以下顺序：

1. 明确输入、输出、约束和验收样例。
2. 把业务描述抽象成数据形状和变换链。
3. 按函数职责选择组件，而不是按案例标题找公式。
4. 从内到外构造公式，并用 `LET` 命名中间结果。
5. 针对 Excel/WPS 环境处理兼容性。
6. 写入后回读结果，检查形状、边界和错误值。

知识库检索到的公式或案例片段只能用于确认某个函数的语义、兼容性或边界条件，不能直接视为当前任务答案。

## 一、先建立任务契约

生成公式前，先把用户需求整理成四类信息。

### 1. 输入

- 数据源区域及所在工作表。
- 每列字段含义、数据类型和表头位置。
- 一行代表什么粒度，例如订单、人员、日期或汇总组。
- 是否存在空行、合并单元格、重复记录、文本数字或混合日期。

### 2. 输出

- 输出是单值、一列、多列、二维矩阵还是可变长度明细。
- 是否需要表头、排序、去重、组内序号或汇总行。
- 输出方向是纵向溢出、横向溢出还是固定单元格。

### 3. 约束

- 使用 Excel 还是 WPS，动态数组功能是否开启。
- 能否使用 `GROUPBY`、`PIVOTBY`、正则函数等环境相关能力。
- 目标区域是否有足够溢出空间。
- 是否必须使用单条公式，还是允许辅助列或辅助区域。

### 4. 验收

- 用户样例是完整结果还是局部规律样例。
- 空值、零、重复键、无匹配、非法日期等边界应得到什么结果。
- 数值精度、文本连接符、排序方向和错误值处理要求。

信息不足时，优先读取工作簿确认；不要从相似案例猜列含义或输出形状。

## 二、把业务问题抽象为数据形状

公式设计首先处理形状，而不是函数名。

### 常见输入形状

- 标量：一个阈值、日期或查找键。
- 向量：一列名称、数量、日期或状态。
- 记录表：每行是一条业务记录，多列共同描述它。
- 宽表：同一指标分散在多列或多个工作表。
- 层级表：上级值只写一次，下方空白表示继承。
- 区间表：每行给出开始值和结束值。

### 常见输出形状

- 标量聚合：合计、均值、计数、最大值。
- 等长映射：输入每行对应一个输出。
- 分组结果：每个唯一键对应一行或一列。
- 展开明细：一行按数量或日期范围变成多行。
- 重塑结果：宽表转长表、长表转矩阵、行列旋转。
- 笛卡尔组合：两组数据生成所有配对。

先写清楚“输入形状 -> 输出形状”。如果形状判断错误，即使局部函数语法正确，最终公式也会溢出方向错误、重复计算或尺寸不一致。

## 三、通用变换链

大多数复杂公式都可以拆成以下流水线，不要求每一步都出现：

`规范化 -> 筛选 -> 映射 -> 分组 -> 聚合 -> 重塑 -> 展示`

### 1. 规范化

把源数据整理成后续函数能稳定消费的数组。

- 去空行：`FILTER(data,key<>"")`
- 忽略空项并转一列：`TOCOL(data,1)`
- 合并单元格值向下继承：`SCAN("",col,LAMBDA(prev,x,IF(x="",prev,x)))`
- 文本拆分：`TEXTSPLIT`、`TEXTBEFORE`、`TEXTAFTER`
- 错误值收敛：只在明确知道错误含义时使用 `IFNA` 或 `IFERROR`

规范化应保留业务含义。不要为了让公式运行而把所有空值改成零，也不要在不确认影响时静默删除记录。

### 2. 筛选

筛选负责保留满足条件的记录，不负责聚合。

```excel
=FILTER(data,(condition1)*(condition2),"")
```

- 多条件“且”使用布尔数组相乘。
- 多条件“或”使用布尔数组相加后判断大于零。
- 筛选区域与条件区域必须同高或同宽。
- 无结果时明确返回空文本或业务指定值。

### 3. 映射

映射表示每个输入元素独立计算一个结果。

```excel
=MAP(items,LAMBDA(x,calculation(x)))
```

适合逐项分类、查找、文本解析和独立计算。如果当前行结果依赖上一行状态，就不应使用纯 `MAP`，应改用 `SCAN`。

### 4. 分组与聚合

分组先确定键，再对每个键计算值。

通用结构：

```excel
=LET(
  k,FILTER(key,key<>""),
  v,FILTER(value,key<>""),
  u,UNIQUE(k),
  HSTACK(u,MAP(u,LAMBDA(x,aggregate(FILTER(v,k=x)))))
)
```

如果环境支持且参数语义已确认，可用 `GROUPBY` 简化。聚合函数由目标决定：求和用 `SUM`，计数用 `ROWS` 或条件计数，文本合并用 `TEXTJOIN`，统计分析按业务口径选择均值、中位数等。

### 5. 状态扫描

`SCAN` 用于当前结果依赖前序状态的任务：向下填充、累计值、组内编号、连续段识别、自定义排序编号。

```excel
=SCAN(initial,array,LAMBDA(state,current,next_state))
```

设计 `SCAN` 时先定义“状态是什么”。状态可以是上一个非空值、累计和、当前组号或组内计数。不要只因为案例使用了 `SCAN` 就套用；没有前序依赖时使用 `MAP` 更清晰。

### 6. 归约与动态拼接

`REDUCE` 适合遍历项目并逐步累积一个可能改变尺寸的结果，例如跨表堆叠、按数量展开、按日期区间展开。

```excel
=DROP(REDUCE(seed,items,LAMBDA(acc,x,VSTACK(acc,build_rows(x)))),1)
```

使用哨兵种子时，最后必须用 `DROP` 删除。若每项输出尺寸固定，优先考虑 `MAP`；只有需要累积或变长拼接时才使用 `REDUCE`。

### 7. 重塑与展示

- 纵向拼接：`VSTACK`
- 横向拼接：`HSTACK`
- 二维转一列/一行：`TOCOL` / `TOROW`
- 一维重新分行/分列：`WRAPROWS` / `WRAPCOLS`
- 选列与换序：`CHOOSECOLS`
- 行列互换：`TRANSPOSE`
- 排序：`SORT` / `SORTBY`
- 长表转矩阵：`PIVOTBY` 或“唯一行键 + 唯一列键 + 二维映射”

重塑前必须计算目标行列数。`HSTACK` 的各数组行数、`VSTACK` 的各数组列数不一致时会产生补位错误，应在拼接前对齐，而不是事后用 `IFERROR` 把结构问题全部隐藏。

### 8. 控制数组与广播

复杂动态数组公式通常不是直接计算最终值，而是先构造控制数组，再按控制数组取值或重塑：

- 位置数组：`SEQUENCE` 生成行号、列号、日期或重复次数序列。
- 条件掩码：纵向数组与横向数组比较，利用广播生成二维布尔矩阵。
- 分组状态：`SCAN` 生成组号、最近有效值或组内位置。
- 动态索引：`MATCH/XMATCH` 找到每行应读取的列号或来源行号。
- 最终取值：`INDEX`、`CHOOSEROWS`、`FILTER`、`GROUPBY` 或 `PIVOTBY` 消费控制数组。

先检查控制数组的形状和边界，再检查最终结果。这样能把“结果不对”定位到索引、掩码、状态或取值阶段，而不是盲目替换整条公式。

## 四、按职责选择函数

| 职责 | 优先考虑 | 判断条件 |
|---|---|---|
| 命名中间结果 | `LET` | 复杂公式、重复引用或需要解释步骤 |
| 条件取行 | `FILTER` | 输出记录数由条件决定 |
| 唯一键 | `UNIQUE` | 分组、去重、生成维度清单 |
| 精确查找 | `XLOOKUP` | 一对一映射，需处理未匹配 |
| 位置查找 | `XMATCH` + `INDEX` | 需要位置、近似匹配或二维取值 |
| 独立逐项计算 | `MAP` | 每项互不依赖 |
| 有状态逐项计算 | `SCAN` | 当前项依赖之前状态 |
| 累积变长结果 | `REDUCE` | 每轮追加行列或跨区域累积 |
| 生成连续位置 | `SEQUENCE` | 展开、编号、日期序列、矩阵索引 |
| 分组聚合 | `GROUPBY` 或 `UNIQUE + MAP` | 按键产生汇总结果 |
| 透视重塑 | `PIVOTBY` 或二维映射 | 行键和列键共同决定单元格 |
| 文本合并 | `TEXTJOIN` | 多记录合并为一个文本 |
| 文本解析 | `TEXTSPLIT` / 正则函数 | 规则稳定且环境能力明确 |
| 逐行/逐列分析 | `BYROW` / `BYCOL` | 每行或每列先归约成一个控制值 |
| 动态选行 | `CHOOSEROWS` | 已经得到有效来源行号数组 |
| 按列位置取值 | `INDEX` / `CHOOSECOLS` | 列位置固定或可动态计算 |

函数越新不代表方案越好。选择标准依次是：语义正确、环境可用、形状稳定、可读、可验证。

## 五、十二类可组合的问题模式

这些是抽象结构，不是可直接替换地址的案例答案。

### 模式 1：按键汇总

结构：`记录表 -> 唯一键 -> 每键筛选 -> 聚合 -> 拼接结果`

适用于按部门、人员、日期或类别汇总数值/文本。先确认键是否为空、是否需要排序、重复记录是否应重复计入。

### 模式 2：层级值补齐后计算

结构：`稀疏层级列 -> SCAN 补齐 -> 与明细列组合 -> 分组或逐行计算`

适用于合并单元格或只在组首行填写上级字段的数据。补齐是预处理，不能直接把视觉合并区域当作每行都有值。

- 状态路线：`SCAN("",key,LAMBDA(prev,x,IF(x="",prev,x)))`，语义直观，适合一般向下继承。
- 位置路线：先生成行号，把非空键所在行作为断点，再用 `XMATCH(...,-1)` 找当前位置之前最近断点，适合还需要断点位置的任务。

### 模式 3：按数量或区间展开

结构：`每行展开长度 -> 累计边界 -> 生成全局位置 -> 定位来源行 -> 生成组内位置`

数量展开有两条常用路线：

- 累计边界路线：用 `SCAN` 生成累计次数，`XMATCH` 把全局位置映射回来源行。适合总展开量较大、需要组内序号的场景。
- 广播索引路线：把次数列与横向 `1..最大次数` 比较，满足的位置写来源行号，不满足的位置写 `NA()`；再用 `TOCOL(...,2)` 提取有效行号并交给 `CHOOSEROWS`。适合重复整行。

```excel
=LET(data,source,n,repeat_count,rowId,SEQUENCE(ROWS(data)),idx,IF(n>=SEQUENCE(,MAX(n)),rowId,NA()),CHOOSEROWS(data,TOCOL(idx,2)))
```

区间展开与数量展开本质相同：先把每行的区间长度转换为展开次数，再生成日期/编号。必须处理零、负数、空值、结束值小于开始值和总长度为零。

### 模式 4：宽表与长表互转

宽转长：识别维度列和重复指标列，生成列标签与行标签，再用 `TOCOL/TOROW/HSTACK` 组装。

长转宽：先生成唯一行键和列键，再按两者组合聚合；环境允许时用 `PIVOTBY`，否则用二维 `MAKEARRAY/MAP` 或查找聚合。

### 模式 5：跨区域或跨表合并

结构：`获取各区域 -> 去空/统一字段 -> VSTACK -> 按业务键或行位置聚合 -> 可选筛选、排序、去重`

合并前先验证字段顺序和数据类型一致。表布局不同不能直接堆叠，应先 `CHOOSECOLS` 对齐。

- 按业务键聚合：各表记录顺序可能不同，必须保留产品代码、人员、日期等真实键，再用 `GROUPBY` 或 `UNIQUE + MAP/FILTER` 汇总。
- 按行位置聚合：只有所有分表行数和记录顺序完全一致时，才能对堆叠后行号按单表行数取模，恢复原始行位置并分组求和。

```excel
=LET(all,VSTACK(source1,source2),n,ROWS(source1),rowKey,MOD(SEQUENCE(ROWS(all))-1,n)+1,DROP(GROUPBY(rowKey,all,SUM,0,0),,1))
```

行位置方案必须验证每张表的首列键逐行一致；只验证行数相同不够。

### 模式 6：文本解析与区间判断

结构：`标准化文本 -> 提取片段 -> 转换类型 -> 业务判断 -> 展示`

正则适合稳定模式；分隔符明确时优先使用 `TEXTSPLIT/TEXTBEFORE/TEXTAFTER`。区间判断要明确边界是否包含，文本日期或数字必须先转换后比较。

### 模式 7：连续段、组内编号和自定义顺序

结构：`识别边界 -> SCAN 维护组状态 -> 生成组号/组内号 -> 排序或汇总`

先定义何时开启新组，例如键变化、日期断档、遇到非空标题或识别到特定文本结构。常用步骤是：

1. `MAP/BYROW` 生成每行是否为组起点的 0/1 标记。
2. 第一次 `SCAN` 累计起点标记，得到组号。
3. 第二次 `SCAN` 在起点处重置、组内递增，得到组内位置。
4. 用 `PIVOTBY` 或 `INDEX` 把“组号 × 组内位置”重塑为二维结果。

状态定义不清时，不要直接抄已有 `SCAN` 公式。

### 模式 8：组合生成

结构：`纵向维度 × 横向维度广播 -> 展平组合 -> 按字段分隔符还原列`

两个维度可直接让纵向数组与 `TOROW` 后的横向数组广播。三个及以上维度可把“加入一个新维度”定义为 `REDUCE` 的单轮操作，每轮生成当前组合与新维度的笛卡尔积。

```excel
=LET(combined,REDUCE(base,SEQUENCE(COLUMNS(dimensions)),LAMBDA(acc,c,TOCOL(acc&delimiter&TOROW(INDEX(dimensions,,c),1)))),TEXTSPLIT(TEXTJOIN(recordDelimiter,,combined),delimiter,recordDelimiter))
```

输出行数是各集合长度之积，写入前先评估规模，避免生成超大溢出数组。字段内容可能包含分隔符时，不能使用文本拼接法，应改用索引法组装。

### 模式 9：每行动态读取不同列

结构：`逐行识别目标表头位置 -> 向下继承最近列号 -> 行号数组与列号数组共同传给 INDEX`

适用于多个纵向数据块字段位置不同、表头只在块首出现的工作表。`BYROW` 负责把每行归约为“目标字段所在列号”；找不到时返回错误；`SCAN` 继承最近成功识别的列号；最后逐行取值。

关键验证：每个数据块必须先出现可识别表头；同一行存在多个同名字段时必须明确取第几个；不要用 `IFERROR` 把第一个数据块之前的缺失列号静默变成错误数据。

### 模式 10：可选条件与模糊查询

结构：`全 1 基准条件 -> 每个非空输入生成条件数组 -> 条件相乘 -> FILTER`

```excel
=LET(data,source,ones,SEQUENCE(ROWS(data))^0,FILTER(data,IF(param1="",ones,condition1)*IF(param2="",ones,condition2)*ISNUMBER(SEARCH(keyword,textColumn))))
```

空参数返回全 1，表示不限制；非空参数才执行精确、模糊或区间判断。文本包含查询优先考虑 `SEARCH`（不区分大小写）或 `FIND`（区分大小写）。日期上下限必须先统一为真实日期值。

### 模式 11：组内密集排名

结构：`MAP 当前键和当前值 -> FILTER 组内值 -> UNIQUE 去重 -> 统计大于等于当前值的档位数`

```excel
=MAP(group,value,LAMBDA(g,v,SUM(--(UNIQUE(FILTER(value,group=g))>=v))))
```

该结构生成降序密集排名：同值同名次，下一档不跳号。若要求竞赛排名（下一名跳号），不能先 `UNIQUE`；若要求升序排名，应把比较方向改为小于等于。

### 模式 12：模板参数逐项替换

结构：`查找模板 -> 生成参数序号 -> REDUCE 逐轮 SUBSTITUTE -> 返回最终文本`

```excel
=LET(template,XLOOKUP(key,templateKey,templateText),replace,LAMBDA(text,args,REDUCE(text,SEQUENCE(ROWS(args)),LAMBDA(acc,i,SUBSTITUTE(acc,"{"&i&"}",INDEX(args,i)))))),replace(template,arguments))
```

适用于占位符数量随模板变化的物料编码、描述生成和批量文本装配。必须验证占位符编号连续、参数数量足够，并区分“缺少参数”和“参数本身为空”。

## 六、从内到外构造公式

复杂公式按以下顺序搭建：

1. 单独验证源范围和类型。
2. 单独写规范化表达式，确认行数。
3. 单独写关键变换，确认中间数组形状。
4. 加入分组、累计或重塑逻辑。
5. 最后用 `HSTACK/VSTACK` 组装展示结果。
6. 用 `LET` 给每个稳定中间结果命名。

示意骨架：

```excel
=LET(
  raw,source_range,
  clean,FILTER(raw,keep_condition),
  keys,TAKE(clean,,1),
  values,TAKE(clean,,-1),
  transformed,transformation(keys,values),
  transformed
)
```

变量名应表达角色，如 `keys`、`values`、`groups`、`positions`、`result`，不要使用无意义的 `a`、`b`、`x1` 堆叠复杂逻辑。

### 大数组分块处理

当一次 `TEXTJOIN/TEXTSPLIT`、正则解析或复杂 `MAP` 会处理数万行时，把数据按固定大小分块：

1. 用 `DROP(source,blockIndex*blockSize)` 定位块起点。
2. 用 `TAKE(...,blockSize)` 取得当前块，最后一块自然取得剩余行。
3. 用一个 `LAMBDA` 定义单块变换。
4. 用 `REDUCE + VSTACK` 累积所有块。

块数应按“首块 + 后续块”统一计算，避免整除时多取空块或非整除时漏掉尾块。分块只解决计算规模，不应改变最终行序和字段语义。

## 七、Excel/WPS 兼容性决策

### 通用规则

- 使用英文函数名和半角符号。
- 参数分隔符默认逗号；若本机区域设置要求分号，再统一转换。
- 动态数组公式只写入锚点，不要填满预计溢出区。
- 避免无边界整列动态计算，优先使用实际数据范围。
- 避免让合并单元格直接参与数组计算。

### 需要区分环境的能力

- 正则：WPS 常用 `REGEXP`，新版 Excel 使用 `REGEXEXTRACT/REGEXTEST/REGEXREPLACE`，语法不能直接假定相同。
- `MAP/LAMBDA` 中的区域引用：WPS 中变量作为区域端点时可能需要写成 `(x):A8`，Excel 常见写法是 `x:A8`。
- `GROUPBY/PIVOTBY`：只有在当前设置允许且实际环境支持时使用；否则退回 `UNIQUE + MAP/FILTER` 或二维映射。
- WPS 专有函数：若使用 `REPTARRAY` 等单边函数，必须同时判断是否需要通用替代方案。

兼容性不是在答案末尾加一句“可能不支持”，而是在函数选型阶段就选择正确路线。

## 八、失败诊断顺序

公式失败时按错误类型定位，不要立刻换成另一个案例公式。

### `#SPILL!`

1. 检查目标溢出区域是否被占用。
2. 检查是否位于表格对象、合并单元格或不允许溢出的区域。
3. 计算预期输出尺寸，确认方向正确。

### `#VALUE!`

1. 检查数组尺寸是否一致。
2. 检查文本数字、文本日期和空值是否参与算术。
3. 检查 `LAMBDA` 返回的是标量还是数组，是否符合外层函数要求。

### `#N/A`

1. 区分“业务上无匹配”和“查找键类型不一致”。
2. 检查前后空格、数字与文本格式、近似匹配方向。
3. 只有业务允许无匹配时才返回空白，不要全局吞错。

### `#REF!`

1. 检查跨表引用、动态范围和溢出引用是否仍存在。
2. 检查索引位置是否越界。
3. 检查删除或移动列后引用是否失效。

### 结果能算但不正确

1. 对比输入粒度和输出粒度，排查重复聚合。
2. 检查分组键是否漏列或多列。
3. 检查边界是否含首尾、排序是否影响近似查找。
4. 用最小反例验证：重复键、空值、零数量、无匹配和最后一组。

## 九、验证闭环

公式写入后必须验证，不以“成功写入”代替“结果正确”。

### 结构验证

- 输出行列数符合预期。
- 表头没有重复，溢出方向正确。
- 拼接区域没有因尺寸不一致产生补位错误。

### 内容验证

- 完整样例逐项一致。
- 局部样例对应行一致，其余行遵循同一规律。
- 聚合结果可用原始数据手工核对至少一组。

### 边界验证

- 空值、零、重复键、无匹配、首组和末组结果合理。
- 日期区间的包含关系正确。
- 动态数组无 `#SPILL!/#VALUE!/#N/A/#REF!`。

### 环境验证

- 公式在当前 Excel/WPS 环境中实际计算。
- 未支持函数应在回读时暴露为 `#NAME?`，再切换兼容方案。
- 若全局关闭动态数组，目标区域中每个公式必须独立计算，不依赖 spill。

## 十、回答与执行准则

面对新问题时，先形成以下内部解题摘要，再生成公式：

1. 输入和输出形状。
2. 关键业务键与粒度。
3. 所需变换链。
4. 每一步选择的函数职责。
5. Excel/WPS 兼容路线。
6. 需要验证的边界。

禁止以下做法：

- 仅因为字段名或样例值相似就复制某个案例公式。
- 未读取当前数据就替换案例中的区域地址。
- 用 `IFERROR` 掩盖尺寸、类型或引用错误。
- 为了使用高级函数而增加不必要的嵌套。
- 公式失败后无分析地轮换多个案例写法。

正确做法是：从当前任务事实出发，按通用变换链组合公式；案例只用于证明某个局部技巧可行，最终答案必须由当前数据结构和验收要求推导出来。
