# WPS 中 LAMBDA 变量加括号的用法

## 场景

在 WPS 表格中使用 `MAP + LAMBDA` 遍历单元格区域时，如果变量 `x` 要参与区域引用拼接，比如 `x:A8`，公式可能报错：

```text
公式中有多余的运算数据
```

原因是 WPS 可能把 `x` 当成单元格的“值”，而不是单元格“引用”。这时给变量加括号，写成 `(x):A8`，通常可以让公式正常执行。

这个写法适合需要兼容 WPS 和 Excel 的动态数组公式。

## 例子

需求：A 列有快递单号，中间有空行，需要把空行用下方第一个非空单号补齐。

原始数据：

| 行号 | A列：快递单号 |
|---|---|
| 2 | jd01 |
| 3 |  |
| 4 |  |
| 5 | sf66 |
| 6 |  |
| 7 |  |
| 8 | yt88 |

目标结果：

| 行号 | 填充后 |
|---|---|
| 2 | jd01 |
| 3 | sf66 |
| 4 | sf66 |
| 5 | sf66 |
| 6 | yt88 |
| 7 | yt88 |
| 8 | yt88 |

原始思路：从当前行到最后一行，忽略空值，取第一个非空值。

```text
A2:A8
A3:A8
A4:A8
A5:A8
...
```

在 Excel 中可用：

```excel
=MAP(A2:A8,LAMBDA(x,TAKE(TOCOL(x:A8,1),1)))
```

在 WPS 中更稳的写法：

```excel
=MAP(A2:A8,LAMBDA(x,TAKE(TOCOL((x):A8,1),1)))
```

记法：

```text
当 LAMBDA 变量要当作区域起点或终点时，WPS 中优先写成 (x)
```

# VSTACK 合并多个工作表

来源：https://mp.weixin.qq.com/s/Spov-zuysns2OiyQpz2kgw

## 场景

多个工作表的字段和布局完全一致，需要把它们纵向合并到一张汇总表里。

如果还要去掉空白行，或者只保留某个地区的数据，可以在 `VSTACK` 外层套 `FILTER`。

## 例子

表1：

| 地区 | 姓名 | 数量 |
|---|---|---:|
| 河北 | 张三 | 10 |
| 山东 | 李四 | 20 |

表2：

| 地区 | 姓名 | 数量 |
|---|---|---:|
| 河北 | 王五 | 30 |
| 河南 | 赵六 | 40 |

表3：

| 地区 | 姓名 | 数量 |
|---|---|---:|
| 浙江 | 小周 | 50 |
| 河北 | 小钱 | 60 |

目标结果：

| 地区 | 姓名 | 数量 |
|---|---|---:|
| 河北 | 张三 | 10 |
| 山东 | 李四 | 20 |
| 河北 | 王五 | 30 |
| 河南 | 赵六 | 40 |
| 浙江 | 小周 | 50 |
| 河北 | 小钱 | 60 |

合并 `表1` 到 `表3` 的数据：

```excel
=VSTACK(表1:表3!A2:C10)
```

去掉合并结果中的空白行：

```excel
=LET(a,VSTACK(表1:表3!A2:C10),FILTER(a,CHOOSECOLS(a,1)<>""))
```

添加标题行：

```excel
=VSTACK(表1!A1:C1,LET(a,VSTACK(表1:表3!A2:C10),FILTER(a,CHOOSECOLS(a,1)<>"")))
```

只保留地区为“河北”的记录：

```excel
=VSTACK(表1!A1:C1,LET(a,VSTACK(表1:表3!A2:C10),FILTER(a,(CHOOSECOLS(a,1)<>"")*(CHOOSECOLS(a,1)="河北"))))
```

# HSTACK + TOCOL + WRAPROWS 转层级名单

来源：https://mp.weixin.qq.com/s/K000UW7dECYY_okLBGSsNQ

## 场景

原始表是普通一维名单：组别、姓名、籍贯各占一列。

目标是转成层级展示：组名单独占一行，下面依次展示组员姓名和籍贯。

## 例子

原始数据：

| 组别 | 姓名 | 籍贯 |
|---|---|---|
| A组 | a | 广州 |
| A组 | b | 深圳 |
| B组 | c | 上海 |
| B组 | d | 北京 |

目标结果：

| 层级列1 | 层级列2 |
|---|---|
| A组 |  |
| a | 广州 |
| b | 深圳 |
| B组 |  |
| c | 上海 |
| d | 北京 |

公式：

```excel
=UNIQUE(WRAPROWS(TOCOL(IFNA(HSTACK(A2:A7,"",B2:C7),"")),2))
```

逻辑：

```text
HSTACK：把“组别 + 空列 + 姓名籍贯”横向拼起来
IFNA：把拼接产生的 #N/A 清成空白
TOCOL：拉平成单列
WRAPROWS：每 2 个元素换成一行
UNIQUE：去掉重复的“组别 + 空白”行
```

# SCAN 处理合并单元格工资均摊

来源：https://mp.weixin.qq.com/s/3Fkzsz0DrFIU9R1uJRtJ5w

## 场景

工资表中有合并单元格，例如 `B2:B4` 合并显示 `48`，代表这 3 个人共同分摊 48。

目标是自动算出每个人应分摊的金额。

## 例子

原始数据：

| 姓名 | B列：合并单元格工资 |
|---|---:|
| 张三 | 48 |
| 李四 |  |
| 王五 |  |
| 赵六 | 50 |
| 钱七 |  |
| 孙八 | 15 |

说明：

```text
B2:B4 合并显示 48
B5:B6 合并显示 50
B7 单独显示 15
```

目标结果：

| 姓名 | 分摊金额 |
|---|---:|
| 张三 | 16 |
| 李四 | 16 |
| 王五 | 16 |
| 赵六 | 25 |
| 钱七 | 25 |
| 孙八 | 15 |

思路：

```text
1. 用 SCAN 给每一行生成组号
2. 用 GROUPBY 统计每组人数
3. 再用 SCAN 把合并单元格金额向下填充
4. 金额 / 组内人数 = 每人分摊金额
```

公式：

```excel
=LET(a,SCAN(0,B2:B7,LAMBDA(x,y,IF(y<>"",x+1,x))),b,GROUPBY(a,a,COUNT,0,0),c,SCAN(0,B2:B7,LAMBDA(x,y,IF(y<>"",y,x))),c/VLOOKUP(a,b,2,0))
```

例子结果：

```text
48 / 3 = 16
50 / 2 = 25
15 / 1 = 15
```

# 连续数字压缩成区间

来源：https://mp.weixin.qq.com/s/RovNzsPKLJSweJBcD1Qf_g

补充来源：https://mp.weixin.qq.com/s/1-T_4Z_oy05jFkbgvCTRfw

## 场景

单元格里有一串用逗号分隔的数字，需要把连续数字压缩成区间。

例如：

```text
1,3,4,5,9,10,13
```

转换成：

```text
1, 3-5, 9-10, 13
```

## 例子

原始数据：

| 单元格 | 原始数字 |
|---|---|
| A3 | 1,3,4,5,9,10,13 |

目标结果：

| 单元格 | 压缩结果 |
|---|---|
| B3 | 1, 3-5, 9-10, 13 |

公式：

```excel
=LET(a,TEXTSPLIT(A3,,",")*1,b,a-SEQUENCE(ROWS(a)),ARRAYTOTEXT(DROP(GROUPBY(b,a,LAMBDA(x,@x&IF(ROWS(x)=1,,-MAX(x))),0,0),,1)))
```

关键技巧：

```text
连续数字 - 位置序号 = 相同差值
```

例如：

```text
3-2=1
4-3=1
5-4=1
```

所以 `3,4,5` 会被分到同一组，再用 `GROUPBY` 聚合成 `3-5`。

# SCAN + GROUPBY 整理物流报表

来源：https://mp.weixin.qq.com/s/oqP5D7MgqlpEtW0fuP5tvQ

## 场景

物流报表中，物流公司和收件人只在首次出现时填写，下面同组记录留空，中间还夹杂空白行。

目标是整理成规范一维表：

```text
收件人 | 物流公司 | 物流单号汇总
```

同一个收件人的多个物流单号需要合并到同一行，用逗号隔开。

## 例子

原始数据：

| 物流公司 | 收件人 | 物流单号 |
|---|---|---|
| 极兔 | a | JT001 |
|  |  | JT002 |
|  | b | JT003 |
|  |  |  |
| 顺丰 | c | SF001 |
|  |  | SF002 |
|  | d | SF003 |

目标结果：

| 收件人 | 物流公司 | 物流单号汇总 |
|---|---|---|
| a | 极兔 | JT001, JT002 |
| b | 极兔 | JT003 |
| c | 顺丰 | SF001, SF002 |
| d | 顺丰 | SF003 |

先提取并调整列顺序：

```excel
=CHOOSECOLS(A1:B10,2,1)
```

`TRANSPOSE` 的作用：

```text
TRANSPOSE 负责转置数组方向
先把纵向两列转成横向两行，方便 SCAN 按顺序填充空白
SCAN 填充完成后，再用 TRANSPOSE 转回纵向表格
```

用 `SCAN` 填充空白：

```excel
=TRANSPOSE(SCAN(,TRANSPOSE(CHOOSECOLS(A1:B10,2,1)),LAMBDA(x,y,IF(y="",x,y))))
```

最终用 `GROUPBY` 按“收件人 + 物流公司”分组，并合并物流单号：

```excel
=GROUPBY(TRANSPOSE(SCAN(,TRANSPOSE(CHOOSECOLS(A1:B10,2,1)),LAMBDA(x,y,IF(y="",x,y)))),C1:C10,ARRAYTOTEXT,3,0,,C1:C10<>"")
```

记法：

```text
TRANSPOSE 负责调整数组方向，配合 SCAN 完成填充
SCAN 负责补齐空白
GROUPBY 负责按人和物流公司分组
ARRAYTOTEXT 负责把多个单号合并成文本
```

# SCAN + HSTACK + GROUPBY 合并单元格多条件汇总

来源：https://mp.weixin.qq.com/s/b5jk3n3zES4onmczvThDRg

## 场景

客户列使用了合并单元格，导致每组只有第一行有客户名。现在需要按“客户 + 区分”汇总数量和金额。

不想取消合并单元格，也不想加辅助列，可以先用 `SCAN` 补全客户名，再用 `HSTACK` 构建复合分组，最后用 `GROUPBY` 求和。

## 例子

原始数据：

| 客户 | 类别 | 区分 | 数值 |
|---|---|---|---:|
| a | 甲 | 数量 | 10 |
|  | 甲 | 金额 | 400 |
| b | 乙 | 数量 | 20 |
|  | 乙 | 金额 | 500 |
|  | 丙 | 数量 | 30 |
|  | 丙 | 金额 | 700 |
| a | 丁 | 数量 | 15 |
|  | 丁 | 金额 | 900 |

目标结果：

| 客户 | 区分 | 合计 |
|---|---|---:|
| a | 数量 | 25 |
| a | 金额 | 1300 |
| b | 数量 | 50 |
| b | 金额 | 1200 |

公式：

```excel
=GROUPBY(HSTACK(SCAN(0,A2:A9,LAMBDA(x,y,IF(y<>"",y,x))),C2:C9),D2:D9,SUM,0,0)
```

记法：

```text
SCAN 负责把合并单元格客户名向下补齐
HSTACK 负责把“客户 + 区分”拼成复合分组字段
GROUPBY 负责按复合字段汇总数值
```

# REDUCE 宽表区县转一维长表

来源：https://mp.weixin.qq.com/s/PWoVAsX4BDraZhLSr2SDyQ

## 场景

现有数据是“省 + 地区 + 多列区县”的宽表，需要恢复成标准一维长表。

同时，目标表中相同省份只在第一次出现时显示，后续重复省份留空。

## 例子

原始数据：

| 省 | 地区 | 区县1 | 区县2 | 区县3 |
|---|---|---|---|---|
| 河北 | 保定 | 清苑 | 满城 |  |
| 河北 | 石家庄 | 长安 | 桥西 | 新华 |
| 山西 | 太原 | 小店 | 迎泽 |  |

目标结果：

| 省 | 地区 | 区县 |
|---|---|---|
| 河北 | 保定 | 清苑 |
|  | 保定 | 满城 |
|  | 石家庄 | 长安 |
|  | 石家庄 | 桥西 |
|  | 石家庄 | 新华 |
| 山西 | 太原 | 小店 |
|  | 太原 | 迎泽 |

公式：

```excel
=LET(a,REDUCE({"省","地区","区县"},A3:A5,LAMBDA(x,y,VSTACK(x,IFNA(HSTACK(y,OFFSET(y,,1),TOCOL(LET(a,OFFSET(y,,2,,3),IF(a="",0/0,a)),2)),"")))),HSTACK(REDUCE(,TAKE(a,,1),LAMBDA(x,y,VSTACK(x,IF(ISNUMBER(MATCH(y,x,0)),"",y)))),TAKE(a,,-2)))
```

记法：

```text
OFFSET 负责从当前省份单元格横向取地区和区县区域
TOCOL 负责把横向区县转成纵向区县
REDUCE 负责逐行处理并不断 VSTACK 追加
第二个 REDUCE 负责压制重复省份，只保留首次出现
```

# SCAN + PIVOTBY 一维长表转二维宽表

来源：https://mp.weixin.qq.com/s/n8xd3HUI5iQ2jHiW4m0FAw

## 场景

原始表中省份和地区存在“省略重复项”的写法：同组只有第一行写省份、地区，后面留空。

目标是把它整理成“省份 + 地区”为一行，后面横向展开该地区下的多个区县，并自动生成第1列、第2列、第3列。

## 例子

原始数据：

| 省 | 地区 | 区县 |
|---|---|---|
| 河北 | 保定 | 清苑 |
|  |  | 满城 |
|  | 石家庄 | 长安 |
|  |  | 桥西 |
|  |  | 新华 |
| 山西 | 太原 | 小店 |
|  |  | 迎泽 |

目标结果：

| 省 | 地区 | 1 | 2 | 3 |
|---|---|---|---|---|
| 河北 | 保定 | 清苑 | 满城 |  |
| 河北 | 石家庄 | 长安 | 桥西 | 新华 |
| 山西 | 太原 | 小店 | 迎泽 |  |

公式：

```excel
=PIVOTBY(TRANSPOSE(SCAN(0,TRANSPOSE(A2:B8),LAMBDA(x,y,IF(y=0,x,y)))),SCAN(0,B2:B8,LAMBDA(x,y,IF(y>0,1,x+1))),C2:C8,CONCAT,,0,,0)
```

记法：

```text
TRANSPOSE 负责把纵向数据转横向，再转回纵向
SCAN 负责补齐省份和地区空白
第二个 SCAN 负责给每个地区内的区县生成横向序号
PIVOTBY 负责按“省份 + 地区”和序号交叉透视
```

适合“每个合并块只在首行显示对应数据”的场景。

左侧数据：

| 分层 | 进尺 | 孔深 |

| SS | SD | VB |

公式：


# SCAN + GROUPBY 合并单元格组内减法

来源：https://mp.weixin.qq.com/s/QGyKiPU_1xZPtcXkO1-KFQ

## 场景

A列是合并单元格里的被减数，B列是同组内多个减数。需要在每组首行计算：

```text
被减数 - 本组所有减数之和
```

组内其他行保持空白。

## 例子

原始数据：

| 被减数 | 减数 |
|---:|---:|
| 10 | 6 |
|  | 2 |
| 20 | 3 |
|  | 5 |
|  | 7 |
| 30 | 4 |
|  | 6 |
|  | 8 |
|  | 1 |

目标结果：

| 被减数 | 减数 | 差值 |
|---:|---:|---:|
| 10 | 6 | 2 |
|  | 2 |  |
| 20 | 3 | 5 |
|  | 5 |  |
|  | 7 |  |
| 30 | 4 | 11 |
|  | 6 |  |
|  | 8 |  |
|  | 1 |  |

公式：

```excel
=IF(A2:A10,LET(a,SCAN(0,A2:A10,LAMBDA(x,y,IF(y,x+1,x))),b,GROUPBY(a,B2:B10,SUM,0,0),A2:A10-VLOOKUP(a,b,2,0)),"")
```

记法：

```text

VLOOKUP 负责把每组减数总和匹配回每一行
```

# XLOOKUP + FIND + ^0 自动标注时段

来源：https://mp.weixin.qq.com/s/1oi7iJ3CmLXpeZLlbM6GmA

## 场景

数据源中每个人的工作时段写在一个单元格里，多个时段用逗号连接。

目标是在排班矩阵里，根据姓名和列标题自动判断该时段是否存在，存在就打勾，不存在就留空。

## 例子

数据源：

| 骑手 | 工作时段 |
|---|---|
| a | 早餐,中午 |
| b | 下午,晚上 |
| c | 早餐,夜晚 |

待标注表：

| 骑手 | 早餐 | 中午 | 下午 | 晚上 | 夜晚 |
|---|---|---|---|---|---|
| a |  |  |  |  |  |
| b |  |  |  |  |  |

目标结果：

| 骑手 | 早餐 | 中午 | 下午 | 晚上 | 夜晚 |
|---|---|---|---|---|---|
| a | √ | √ |  |  |  |
| b |  |  | √ | √ |  |

公式，填在 `C2` 后向右向下复制：

```excel
=IFERROR(REPT("√",FIND(C$1,XLOOKUP($B2,$C$7:$C$9,$D$7:$D$9))^0),"")
```

记法：

```text
XLOOKUP 负责按骑手姓名找出该人的时段文本
FIND 负责判断当前列表头是否出现在时段文本中
^0 负责把找到的位置数字统一变成 1
REPT 负责按 1 生成一个 √
IFERROR 负责把找不到时的错误值变成空白
```
