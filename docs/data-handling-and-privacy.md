# 数据处理事实说明

更新时间：2026-07-18

本文是面向工程、运维和安全审查的数据清单，描述当前代码实际处理的数据及已知缺口。它不是经法律审核的最终隐私政策，也不替代客户合同、处理者协议、跨境评估或当地法律要求。正式上线前，项目所有者必须补充主体身份、联系方式、处理目的与法律依据、适用地区、第三方处理者、用户权利和投诉渠道。

## 默认边界

- 桌面端主要数据保存在用户选择的数据目录；新安装默认使用当前 Windows 用户的 `%LOCALAPPDATA%` 隔离目录。
- 模型对话仍会发送到用户配置的当前模型服务商；选择服务商、Base URL 和要发送的内容由用户或部署方负责。
- Provider API Key、远程压缩 API Key、MinerU Token 和自定义请求头在 Electron 主进程中通过 Windows `safeStorage` 加密存储，Renderer 只接收掩码。
- 会话 JSONL/冷归档、state-runtime SQLite 敏感字段与 knowledge 正文/metadata 使用 AES-256-GCM 应用层加密；主密钥由 Electron `safeStorage` 保护，支持设置页显式密钥轮换。
- 高置信凭据会在 OCR、搜索、Embedding 和发票抽取的统一出站检查中被阻断，但该检测不是数据分类或 DLP 的完整替代品。

## 桌面端本地数据

| 数据类别 | 位置或组件 | 用途 | 当前控制与生命周期 |
| --- | --- | --- | --- |
| 设置与服务商配置 | Electron settings store | 模型、主题、数据目录和功能设置 | 秘密字段使用 `safeStorage`；非秘密设置按用户配置保留 |
| 会话与消息 | `sessions/YYYY/MM/DD/*.jsonl`、`sessions/state-runtime/*.db` | 会话恢复、状态、搜索和审计副本 | 用户可按会话删除 JSONL、`.jsonl.gz/.jsonl.zst` 冷归档、SQLite 投影、搜索索引和工具日志；删除任一文件失败时不继续删除数据库投影 |
| 长期记忆与目标 | `sessions/state-runtime/` 下独立 SQLite | 用户偏好、纠错、目标和运行状态 | 支持单条记忆删除、隐私导出与已登记活动根/迁移旧根/应用隐私导出的擦除及删除证明；未登记外部副本、原始附件与身份级/账户级跨系统请求仍不覆盖 |
| 知识库 | `knowledge/knowledge.db` | 本地索引、检索和可选 Embedding | 用户可删除来源索引；删除索引不会删除原始源文件 |
| Office 备份与事务 | `office-backups/`、`office-automation/workflows/`、`office-automation/transactions/` | 回滚、撤销/重做和持久化工作流 | 备份/事务默认保留 30 天，工作流 90 天，并有条目和字节配额；运行中记录受保护；可随当前活动数据根一起擦除 |
| 应用日志 | `logs/` | 故障排查和安全审计 | 写入前递归脱敏并限制单行；默认保留 30 天、30 个文件、100 MiB；可随当前活动数据根一起擦除 |

数据目录迁移会验证并切换核心数据，但使用者应在迁移或卸载前备份所需文件。卸载程序不应被视为已验证的全局数据擦除工具。

## 远程数据流

| 场景 | 可能发送的数据 | 目标 | 条件 |
| --- | --- | --- | --- |
| 模型对话和工具编排 | 用户消息、选择的附件/图像、必要上下文和工具结果 | 用户配置的模型服务商或兼容 Base URL | 使用模型功能时；用户必须了解所选 Provider 的政策 |
| 远程 OCR | 用户明确选择且本地解析未完成的文件 | 配置的 MinerU 服务 | 本地解析未完成、对应服务可用且出站检查通过 |
| 联网搜索 | 查询词 | 配置的搜索服务 | 调用联网搜索且出站检查通过 |
| 知识 Embedding | 知识片段 | 配置的 Embedding/模型服务 | 对应服务可用且出站检查通过 |
| 发票智能抽取 | OCR Markdown 或相关文本 | 当前模型服务商 | 发票抽取被调用且出站检查通过 |
| 应用更新与安装包下载 | 版本、更新请求和常规 HTTP 元数据 | `plugin.shelelove.top` | 检查更新或下载时；不发送 Office 文档内容 |

实际第三方保存时间、训练使用、地域和删除方式由用户配置的服务商决定。企业部署应使用经批准的服务商 allowlist、合同和区域配置，并在调用对应远程能力前向最终用户告知。

## 产品站下载统计

产品站记录下载时间、版本、制品名称、周期化 IP HMAC、最多 200 字符的 User-Agent 和仅保留 Origin 的 Referer。它不保存原始 IP、Referer 路径、查询参数或片段。

- 下载记录默认保留 90 天，可在 1 至 3650 天范围配置。
- IP HMAC 默认每 30 个 UTC 日轮换，可在 1 至 365 天范围配置；不持久化周期密钥。
- 服务启动时及之后每 6 小时清理过期记录；统计或清理失败不得阻塞安装包下载。
- 在线 SQLite 备份默认每天执行并保留 14 份；生产负责人仍需确认 RPO/RTO、异机副本、告警和恢复演练。

周期标识是降低长期关联能力的假名化控制，不等同于匿名化。后台访问日志和反向代理日志还需由生产运维单独设置留存与访问控制。

## 用户控制与当前缺口

当前可用控制包括：删除单个会话及其冷归档、删除单条长期记忆、删除知识来源索引、更换本地数据目录，以及从设置页执行本地数据隐私导出。导出会把应用持有的设置、会话、记忆/目标、知识索引、Office 备份/自动化、日志和临时文件复制到用户选择的空目录；它先暂停会话写入、关闭 SQLite/知识运行时，对核心文件完成逐文件哈希校验后恢复运行，仍可能变化的日志和临时文件按 best-effort 快照复制。API Key、OCR Token 和自定义请求头秘密被省略或掩码。

设置页还提供应用本地数据擦除。用户必须输入精确短语 `ERASE LOCAL DATA`；活动 turn 或数据维护期间拒绝执行。主进程在同一维护窗口内暂停写入并关闭 SQLite/知识运行时后，对已登记活动根、迁移旧根与应用创建的隐私导出副本执行类别化擦除（拒绝符号链接/联接、磁盘根与父子包含关系）；擦除前先预创建 replacement 数据密钥，删除完成后单次原子写将 replacement 升为唯一 active 并清除全部旧密钥材料，全程保持有效 payload protection（无空 keystore 窗口）。删除证明写入 Electron bootstrap/userData 独立审计位置，结果返回摘要（含 installId 摘要与密钥销毁结果）。当前数据根路径会保留以便同位置恢复。目录外原始 Office 文档、附件、知识源文件、运维侧未登记副本与身份级/账户级跨系统请求仍不覆盖。

以下能力尚未形成闭环，不应向用户或客户宣称已经提供：

- 覆盖原始附件、产品站代理日志和外部备份的统一 TTL、删除请求编排与过期策略；
- 按身份处理产品站统计的访问/删除请求；
- 企业密钥托管与云同步（产品为纯本机，无云账号）；
- 经法律审核的隐私通知、处理者清单、跨境机制和数据主体请求流程。

在这些能力完成前，企业上线必须由责任人形成书面风险接受或部署侧补偿控制，并在客户文档中使用与本说明一致的事实口径。

## 运维要求

- 数据目录、产品站数据库、备份和日志只授予专用账户最小权限。
- 不在日志、工单、测试、截图或安全报告中复制真实文档、原始 IP 或秘密。
- 远程服务、Base URL、搜索和 OCR 端点应使用 HTTPS、精确 allowlist 和经批准的合同。
- 数据泄露或凭据暴露按 [安全策略](../SECURITY.md) 私下报告并立即轮换受影响秘密。
- 每次改变数据字段、远程目的地、默认行为、留存或删除语义时，同步更新本文和自动化治理测试。
