# Installer UI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a branded Windows NSIS installer experience named “文格 AI 助手”.

**Architecture:** Keep the existing `electron-builder` + NSIS pipeline. Generate deterministic BMP installer artwork from a local Node script, then wire those resources into `desktop/package.json` without changing app runtime code.

**Tech Stack:** Electron Builder NSIS, Node.js script, BMP binary writer, npm build scripts.

---

## File Structure

- Create `desktop/scripts/generate-installer-assets.cjs`: deterministic local generator for NSIS `installer-sidebar.bmp` and `installer-header.bmp`.
- Create `desktop/build/installer-sidebar.bmp`: 164x314 sidebar artwork generated by the script.
- Create `desktop/build/installer-header.bmp`: 150x57 header artwork generated by the script.
- Modify `desktop/package.json`: change `build.productName`, `build.nsis.shortcutName`, and add `build.nsis.installerSidebar` / `build.nsis.installerHeader`.
- Modify `CHANGELOG.md`: record installer branding implementation.

### Task 1: Add Deterministic Installer Asset Generator

**Files:**
- Create: `desktop/scripts/generate-installer-assets.cjs`

- [ ] **Step 1: Create the generator script**

Use a dependency-free CommonJS script. It writes 24-bit BMP files with simple shape/text-like block marks, avoiding new npm dependencies and binary-only source assets.

Key functions:

- `createCanvas(width, height, bg)`
- `fillRect(canvas, x, y, w, h, color)`
- `drawBmp(canvas, filePath)`
- `drawSidebar(filePath)`
- `drawHeader(filePath)`

- [ ] **Step 2: Run the generator**

Run:

```bash
node desktop/scripts/generate-installer-assets.cjs
```

Expected:

- `desktop/build/installer-sidebar.bmp` exists.
- `desktop/build/installer-header.bmp` exists.
